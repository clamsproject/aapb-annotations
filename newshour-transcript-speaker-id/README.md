# NewsHour Transcript Speaker Identification

## Project Overview

Derived dataset for speaker diarization, identifying which speaker is talking at each time interval in NewsHour broadcasts. This dataset is automatically generated by combining time-synchronized transcript annotations with gold transcripts that contain speaker markers.

**Note:** This is "gold-ish" data—while derived from human-created sources, the derivation process introduces approximations (see Data Quality section below).

## Annotation Guidelines

This dataset is not manually annotated. It is automatically derived from two existing data sources:

1. **Time-sync annotations** (`newshour-transcript-sync/golds/`) - Maps timestamps to character offsets in transcripts
2. **Gold transcripts** (`aapb-collaboration/21/`) - Plain text transcripts with speaker markers

### What to Annotate

* `label` - The normalized name of the speaker for each time interval (spaces replaced with underscores)

### Decisions, Differentiation, and Precision Level

- **Speaker name variation** - Transcripts use varying forms for the same speaker (e.g., "ROBERT MacNEIL" vs "MacNEIL" vs "MR. MacNEIL"). The output preserves these variations as they appear in the source. No normalization or speaker linking is performed.
- **Multi-speaker time frames** - When a time frame spans multiple speakers, two strategies are available:
    - *Majority*: Assigns the entire slice to the speaker with more characters in that range. Simpler output, but loses speaker transition information.
    - *Split*: Divides the time proportionally based on character counts, assuming uniform speaking rate. Preserves transitions but timing is approximate.
- **Section headers** - Text like "Intro" and "News Summary" are excluded as non-speaker content.
- **Annotations in speaker markers** - Bracketed annotations like `[voice-over]` and parenthetical notes like `(D) New York` are stripped from speaker names.

#### Data Quality

**Source data limitations:**

- **Coarse time intervals** - The time-sync annotations (`newshour-transcript-sync`) were created by marking only start points approximately every 10 tokens; end times are inferred from the next segment's start time. This means:
    - Speaker boundaries may fall anywhere within a time frame, not necessarily at frame boundaries.
    - Pauses or silence between segments are not captured—they get absorbed into the preceding speaker's time.
    - The final segment's end time may be particularly unreliable.
- **Unknown speaker annotation guidelines** - The speaker markers in the gold transcripts were added by the original NewsHour production team during transcription. We do not have access to their annotation guidelines, so handling of edge cases (overlapping speech, off-screen speakers, voice-overs, etc.) is unknown.

**Derivation process limitations:**

- **Character-to-time mapping** - Speaker assignments are based on character offsets, not acoustic features. The mapping assumes the transcript accurately reflects what was spoken and when.
- **Uniform speaking rate assumption** - The "split" strategy divides time proportionally by character count, which assumes speakers talk at a constant rate. In reality, speaking rates vary.
- **No speaker verification** - The process trusts the source speaker markers without validation against the audio.

## Data Format and `process.py`

### Source data

This project has no `raw` annotation data. Instead, it derives gold data from:

1. **Time-sync TSV files** (`newshour-transcript-sync/golds/*.tsv`)
    * Fields: `start`, `end`, `alignment-start`, `alignment-end`
    * Maps time intervals to character offset ranges in the transcript

2. **Gold transcript TXT files** (`aapb-collaboration/21/*-transcript.txt`)
    * Plain text with speaker markers in the format `SPEAKER NAME: spoken text...`
    * May include section headers, bracketed annotations, and varying speaker name formats

### [`process.py`](process.py)

The script joins time-sync annotations with speaker information extracted from gold transcripts.

**Dependencies:** Requires `clams-utils` package for shared transcript processing utilities. Install from PyPI via:

```bash
pip install clams-utils
```

**Processing steps:**

1. Parses the gold transcript to identify speaker markers and their character positions using regex patterns NH transscript cleaner module from `clams-utils`
2. Reads the time-sync TSV to get timepoint-to-character alignments
3. For each time frame, finds which speaker span(s) overlap the character range
4. Applies the selected strategy (majority or split) to assign speakers to time intervals
5. Outputs a TSV with speaker assignments

**Usage:**

```bash
python process.py \
    --gold-tsv-dir ../newshour-transcript-sync/golds \
    --gold-transcript-dir /path/to/aapb-collaboration/21 \
    --output-dir ./golds \
    --strategy majority|split
```

### `golds` data

`.tsv` files - Tab-separated values with one row per time interval (more rows with "split" strategy).

* Fields:
    * `start` - Start time in ISO 8601 format (`hh:mm:ss.mmm`)
    * `end` - End time in ISO 8601 format (`hh:mm:ss.mmm`)
    * `label` - Normalized speaker name (spaces replaced with underscores)
